AgentSentinel

Runtime security monitoring for AI agents that use tools.

Microsoft Hackathon · Built by Nihar Joshi

The shift

AI agents no longer just talk — they act.

Modern agents connect to tools through MCP — they read files, call APIs, query databases and send data on their own.
That power is the point — but every tool call is also a way the agent can be tricked or hijacked.
Today this traffic flows completely unmonitored. Nobody is watching what the agent is actually doing.
The problem

Agents trust whatever flows through them.

Prompt injection

Hidden instructions inside a web page or file hijack the agent's goals.

Identity spoofing

A message pretends to come from a trusted agent that was never verified.

Data exfiltration

A secret is read, then quietly shipped out to an external endpoint.

Privilege escalation

A chain of innocent-looking calls adds up to one dangerous action.

The idea

A transparent guard between the agent and its tools.

AgentSentinel sits in the middle as a proxy. It sees every request and every response, scores the risk, and raises alerts — without ever changing how the agent behaves.

What it does

Four detectors, one live trust graph.

Detect

Four engines flag prompt injection, spoofing, exfiltration and privilege-escalation chains in real time.

Map

A trust graph draws the agent, the tools it touched, and external domains — risky links glow red.

Explain

Every alert opens to show which detector fired, its confidence, and the exact evidence.

Persist

Sessions and events are stored, scored and streamed live to the dashboard via SSE.
